**JDK** and **JRE** are both parts of the Java ecosystem, but they serve different purposes.

---

## 🔹 JRE (Java Runtime Environment)

The **JRE** is what you need to **run Java programs**.

* It provides the **Java Virtual Machine (JVM)**, which actually executes Java bytecode.
* Includes core libraries and files needed to run programs.
* **Does NOT include development tools** like compilers.

👉 Think of JRE as:

> “The engine that runs Java applications.”

---

## 🔹 JDK (Java Development Kit)

The **JDK** is used to **develop (create) Java programs**.

* Includes everything in the JRE **plus** development tools.
* Key tool: `javac` (Java compiler) to convert source code into bytecode.
* Also includes debugging tools, documentation tools, etc.

👉 Think of JDK as:

> “A full toolkit for building and running Java applications.”

---

## 🔁 Relationship

* **JDK = JRE + Development Tools**
* **JRE = JVM + Libraries**

---

## 🧠 Simple Analogy

* **JDK** → Kitchen (you can cook food)
* **JRE** → Dining table (you can only eat food)

---

If you're just running Java apps → install **JRE**
If you're coding in Java → install **JDK**
